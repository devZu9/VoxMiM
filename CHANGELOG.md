## [0.7.4] — 2026-07-04

### Исправлено

- **VAD не слышал микрофон** — детектор использовал сырую среднюю квадратов (`mean(s²)`) на микро-кусочках 1-10мс. Реальные значения с микрофона (f32 ~0.001-0.01 RMS) давали энергию 1e-6–1e-4, что было ниже порога 0.005. VAD никогда не видел речи.
  - VAD переведён на RMS (`√mean(s²)`) — как в проекте DJA
  - Пороги пересчитаны под реальный сигнал: aggr 0=0.05, 1=0.03, 2=0.015, 3=0.008 (RMS)
  - В AudioProcessor добавлено накопление чанков до ~100мс перед вызовом VAD — сглаживает микро-провалы внутри речи
- **Двойной дренаж audio_buf при VAD-автостопе** — AudioProcessor сам забирал сэмплы из буфера и отправлял в whisper, после чего `on_stop()` получал пустой буфер. Исправлено: AudioProcessor только сигнализирует остановку, весь дренаж делает `on_stop()`.
- **Stale RecordingResult ломает новую запись** — результат от предыдущей записи приходил во время новой и сбрасывал `state = Idle`. Когда VAD останавливал новую запись, `on_stop()` выходил раньше времени. Исправлено: `on_result()` проверяет `state == Recording` и игнорирует устаревшие результаты.
- **Гонка HOOK_REC в VAD tap-mode** — после VAD-автостопа HOOK_REC оставался `true`, и следующий Insert посылал Stop вместо Start. Исправлено: в VAD-режиме Insert всегда шлёт `StartRecording`, force-stop делает `on_start()`.

### Добавлено

- **`keep_wav: bool`** — настройка сохранения WAV-файлов. Когда включена, WAV-файлы запросов сохраняются в папке `wavs/` рядом с exe с датой в имени (не удаляются).
- **Галочка «Сохранять WAV-файлы»** — на вкладке «Прочее» окна настроек (Fenestra).
- **VAD debug WAV** — при `keep_wav=true` и срабатывании pre-speech таймаута сохраняется `wavs/vad_debug_...wav` с содержимым буфера, который VAD счёл тишиной. Для диагностики.
- **Логи VAD-переходов** — SpeechStart, Silence, pre-speech таймаут с RMS и счётчиками.

### Изменено

- Все WAV-файлы теперь пишутся в `wavs/` рядом с exe (не в TempDir) — независимо от `keep_wav`.

### Версии

- **voxmim**: 0.7.3 → 0.7.4
- **voxmim-settings**: 1.0.3 → 1.0.4

## [0.7.3] — 2026-07-03

### Исправлено

- **Автоповтор Insert в VAD-режиме** — Windows шлёт повторные `WM_KEYDOWN` каждые ~30мс при зажатой клавише, что вызывало бесконечную трель Start/Stop. Добавлен `VAD_KEY_LOCK` — первый `WM_KEYDOWN` обрабатывается, автоповтор игнорируется до `WM_KEYUP`
- **HOOK_REC не сбрасывался после VAD-автостопа** — следующий Tap Insert отправлял `StopRecording` вместо `StartRecording`. Добавлена `reset_recording_state()`, вызываемая в `on_stop()` и `force_stop()`
- **audio_buf не чистился при старте записи** — остатки предыдущей записи могли попасть в новый сеанс. Добавлен `.clear()` в `on_start()`

### Добавлено

- **Pre-speech таймаут** — после Tap Insert VAD ждёт `start_timeout_secs` (2 сек по умолч.) начала речи. Если молчание — запись отменяется без транскрибации
- **`start_timeout_secs` в конфиг** — новое поле `vad.start_timeout_secs` в `VadConfig` (с дефолтом через `serde(default)`)
- **Поле в окне настроек** — на вкладке Recording добавлен ввод «Таймаут ожидания речи»
- **Независимая версия `voxmim-settings`** — `1.0.3`, своя семвер-линия
- **Версия в заголовке окна настроек** — `env!("CARGO_PKG_VERSION")` в title баре

## [0.7.2] — 2026-07-03

### Исправлено

- **Корень проблемы разорванных слов** — whisper server рендерит текст фиксированной шириной (например, 80 символов), из-за чего вставляет `\n` в середине предложения или даже в середине слова. Это `\n` интерпретировался как пробел, разрывая слова. После удаления `\n` из ответа сервера эвристики `space_fixer` стали избыточны — слова перестали разрываться на source-уровне
- **Крэш настроек** — убран субклассинг окна с `WM_NCCALCSIZE`, возвращён `remove_window_caption()` с `EnumWindows` + задержка
- **Окно не поверх всех** — исправлено через CBT-хук (двойная фиксация: стиль + z-order)

### Добавлено

- **Отдельное окно настроек (Fenestra)** — `voxmim-settings.exe` с собственным UI на Fenestra
- **Named Pipe IPC** — связь между `voxmim.exe` и `voxmim-settings.exe` через Win32 Named Pipe
- **Named Mutex** — single instance для окна настроек (``Local\VoxMiMSettingsInstance``)
- **Локализация окна настроек** — `lang/ru.json` + `lang/en.json` через `include_str!`
- **Тёмная тема** — переключение в окне настроек, применяется через `Theme::dark()`
- **Always-on-top** — `WH_CBT` hook с `WS_EX_TOPMOST` + `SetWindowPos(HWND_TOPMOST)` для окна настроек
- **Кастомная кнопка закрытия** — через Fenestra `Msg::Close` → `std::process::exit(0)`
- **`install_topmost_hook()`** — CBT-хук на `HCBT_CREATEWND` + `HCBT_ACTIVATE` для гарантированного «поверх всех»
- **`__run.bat` / `__run_debug.bat`** — `cargo build --workspace`, вывод в консоль через Tee-Object

### Изменено

- **Отступы в окне настроек** — `SP1` (4px) → `SP2` (8px) для gap между элементами
- **Файлы сборки** — `build.rs` копирует `settings/` и `lang/` в `target/debug/`
- **Пользовательский словарь** — `save()` сортирует ключи: ASCII → Кириллица
- **Настройки** — `reload_config()` логирует только изменённые поля (22 поля)

### Удалено

- **Slint UI** — старый `ui/settings.rs` + `ui/settings.slint` (заменён на Fenestra)
- **`image` крейт** из зависимостей `settings` — иконка вшивается через `embed-resource`
- **Мёртвый код** из `text/dictionary.rs`: `Dictionary::path()`, `Dictionary::new()`, `Dictionary::load_lang()`
