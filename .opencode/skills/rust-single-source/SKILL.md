# Rust — Единая точка обращения к данным

## Если проект использует egui / GUI-фреймворк
- Настройки → отдельный `SettingsManager` / `config.rs`
- Цвета и тема → `theme.rs`
- Пути → `dirs::data_dir()`, `dirs::config_dir()` (крейт `dirs`)

## Если проект CLI / без GUI
- Настройки → `confy` или ручной save/load в JSON (крейт `serde` + `serde_json`)
- Пути → `dirs::data_dir()`, `dirs::config_dir()`

## Если проект был TUI (ratatui, cursive)
- Настройки → `confy` или свой config в JSON
- Пути → `dirs::config_dir()` / `dirs::data_dir()`

## Общее для всех
- Пути только через `%USERPROFILE%`, `%LOCALAPPDATA%`, `%APPDATA%` или `dirs` крейт — никаких `C:\...`
- Каждая переменная хранит собственное значение (цвет, путь, число). Нельзя, чтобы одна переменная просто ссылалась на другую:
  - ✅ `editorBg = "#13294B"`, `editorText = "#E0EDFF"`
  - ❌ `editorBg = bg_panel`, `editorText = text_primary`
  - По умолчанию — без взаимных ссылок. Исключение: константы/значения по умолчанию или явный запрос разработчика.
- По возможности не хардкодить переменные и списки — единая точка обращения к данным
