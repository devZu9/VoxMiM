# Rust — API First

Когда пользователь описывает желаемый интерфейс (функции, их сигнатуры, поведение, структуру данных) — реализуй **именно его, один в один**.

## Правила
- Не подменять модель пользователя своей, более сложной архитектурой
- Начинать с простейшей рабочей реализации, как описано
- Добавлять сложность только если пользователь явно попросит

## Пример
Пользователь описал:

```rust
fn cls()
fn print(index: usize, text: &str)
fn scroll(speed: f32)
```

✅ Реализовать `cls()`, `print()`, `scroll()` — три функции, как есть.
❌ Не писать вместо этого абстрактные трейты, обобщённую архитектуру scroller/display/overlay и прочие преждевременные абстракции.

## Почему
- Пользователь продумал модель до тебя. Твоя задача — реализовать, а не перепроектировать.
- Простая реализация работает, её легко менять, в ней меньше багов.
- «Сначала сделай как сказано, потом оптимизируй если попросят».
